How to ‚ùì

  • Visualize 5G measurements taken indoors within a defined environment (referred to as a "room" in the application). A heatmap is generated using data collected from both mobile devices and Raspberry Pi units configured for high-precision measurements.
  • Select which measurement sessions to display by using the checkboxes available in the "Measured Sessions" selector panel.
  • Inspect detailed information for a specific measurement session and optionally enable correction of Pi-based measurements to estimate potential measurement error, using the Raspberry Pi as a reference device. This can be toggled via the slider in the session's popup.
  • Switch between different measurement parameters to display on the map using the dropdown menu located on the left-hand side of the map view.
  • Access session details by clicking on the session markers on the map, which represent the global coordinate origin for each session.
  • Use zoom and pan gestures to navigate and explore the map-based visualization.
  • Double-click on any point of the map to trigger estimations based on the currently available measurement data.
  • Manually edit the global origin coordinates (latitude and longitude) of a session via the tables below if improved accuracy is required for your application.
  • Delete individual measurements or entire measurement sessions to maintain a clean and relevant dataset.
  • Toggle between different base map tile providers via the map settings menu located in the top-right corner of the map interface.
